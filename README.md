MyStockQuant 专业量化策略控制台 V3.0
1. 业务背景与核心价值
本项目是一个专为个人投资者设计的全自动股票技术形态扫描与跟踪系统。其核心价值在于利用量化手段代替人工复盘，在大盘 5000 多只股票中，自动化筛选出符合“主力异动”特征的标的，并通过状态机逻辑跟踪其后续的洗盘与突破过程，旨在捕捉高胜率的起爆点。

系统通过“分级池流转”机制，模拟职业投资者的选股与跟踪流程，将标的分为四个核心观察池：

试盘池：刚发生异动，处于观察期。

回踩池：洗盘中，寻找黄金坑位。

突破池：越过压力位，具备起爆潜力。

废弃池：形态破坏或周期过长，予以自动淘汰。

2. 核心策略逻辑：倍量异动状态机
系统采用严苛的过滤算法，在 bootstrap_pipeline.py 和 strategy_engine.py 中定义了如下核心铁律：

2.1 选股模型：倍量试盘 (V3.0 极客版)
地量横盘 (潜伏期)：

过去 20 个交易日振幅需小于 25%，确保排除高位震荡或趋势明显的股票。

主力试盘 (信号点)：

成交量 $\ge$ 过去 20 日均量的 2.5 倍。

必须是实体涨幅 $> 2%$ 的阳线。

防守位设定：

标杆价：以试盘日最高价（或上影线过长时的实体高点）为压力参考位。

支撑位：以试盘日最低价为生命线。

2.2 状态流转与监控机制 (The State Machine)
股票入池后，会根据每日 K 线走势在四个池子间自动流转：

状态 1 (试盘池)：刚触发倍量信号的标的。

状态 2 (回踩池)：试盘后出现极致缩量（成交量 $\le$ 试盘日 50%），说明主力洗盘且未出货。

状态 3 (突破池)：股价收盘有效站上“标杆价”，视为起爆信号。

状态 99 (废弃池)：一旦收盘跌破支撑位，潜伏超过 25 天未突破，或洗盘期冲高超标杆价 8%（防诱多），系统将自动淘汰。

3. 数据库架构
项目使用 SQLite 数据库 (stock_quant.db)，核心包含以下三张表：

表 1：daily_k_line (底层行情数据)
存储全市场股票的历史及最新 K 线数据。

字段	类型	含义
date	TEXT	交易日期
code	TEXT	6 位股票代码
close	REAL	收盘价（前复权）
volume	REAL	成交量（手）
amount	REAL	成交额（元）
turn	REAL	换手率 (%)
pctChg	REAL	涨跌幅 (%)
表 2：stock_pipeline (策略流水线)
记录当前处于活跃状态的股票及其策略参数。

字段	类型	含义
code	TEXT	股票代码 (主键)
name	TEXT	股票名称
status	INTEGER	状态码 (1:试盘, 2:回踩, 3:突破, 99:废弃)
test_high	REAL	标杆价（晋级参考点）
latest_price	REAL	最新成交价（缓存字段，提升 App 性能）
entry_date	TEXT	进入该状态的时间
update_time	TEXT	最后更新时间
表 3：operation_log (审计日志)
记录系统的每一次自动流转和人工干预，确保每一步调仓都有据可查。

字段	类型	含义
log_date	TEXT	操作日期
log_time	TEXT	具体时间
source	TEXT	触发来源 (自动处理/按钮处理/手工干预)
code	TEXT	相关股票代码
name	TEXT	股票名称
detail	TEXT	变更详情说明
4. 功能模块说明
4.1 Web 总控台 (index.html)
增量同步：一键拉取全市场最新 K 线，运行状态机，实现自动化调仓。

审计日志与上帝模式：按日期分组查看历史流转记录。若认为系统误判，可通过下拉菜单手动修改股票状态并自动写入审计日志。

快捷看图：内置 ECharts，支持在日志界面点击“📊 快捷看图”瞬间调出 K 线分析，无需切换页面。

三重过滤：支持勾选“包含 ST”、“包含创业板”、“包含科创板”，实时影响计数与列表展示。

4.2 后端服务 (server.py)
全局任务互斥锁：确保在执行“增量同步”或“系统重置”时，不会因为重复点击导致数据库死锁或状态混乱。

BFF 模式：针对 Web 端和 App 端提供不同的接口。App 端接口经过数据预装配（如缓存最新价、计算涨幅），实现毫秒级响应。

4.3 定时任务 (auto_task.py)
执行时间：每个交易日 15:35 自动运行（收盘后）。

流程顺序：

update_daily_k_lines()：更新全市场 K 线数据。
run_strategy_engine()：策略引擎执行状态流转。
sync_data_to_app_table()：同步数据到 App 专用视图表。
5. 接口清单及意义
接口路径	方法	动作意义
/api/task_status	GET	系统体检：前端通过此接口判断后台是否在忙，动态置灰操作按钮。
/api/run_strategy	POST	核心动力：手动触发全市场扫描与逻辑判断任务。
/api/pool/{status}	GET	分类查看：根据状态码获取对应的股票清单（Web端格式）。
/api/pool/app/{status}	GET	App 增强视图：获取包含涨跌幅、换手率等实时数据的列表。
/api/kline/{code}	GET	行情调取：为前端 ECharts 提供高精度的历史 K 线数据。
/api/update_status	POST	人工纠偏：上帝模式接口，手动干预状态机并强制记录审计日志。
/api/reset_bootstrap	POST	系统重置：清空当前状态池，基于全量历史数据重新进行雷达回溯。
