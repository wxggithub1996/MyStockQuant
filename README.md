# MyStockQuant 专业量化策略控制台 V3.0

## 📋 项目概述

MyStockQuant 是一个专为个人投资者设计的全自动股票技术形态扫描与跟踪系统。其核心价值在于利用量化手段代替人工复盘，在 A 股市场 5000 多只股票中，自动化筛选出符合“主力异动”特征的标的，并通过“状态机”逻辑跟踪其后续的洗盘与突破过程，捕捉高胜率的起爆点。

系统采用 **倍量试盘 + 缩量洗盘 + 突破确认** 三级状态流转模型，结合 SQLite 本地数据存储与 FastAPI 后端服务，提供 Web 总控台与移动端 App 双端访问能力，并内置定时任务实现全自动无人值守运行。

---

## 🧠 核心策略逻辑：倍量异动状态机

系统采用“分级池流转”机制，模拟职业投资者的选股与跟踪流程，将股票在四个逻辑池中自动迁移。

### A. 选股模型：倍量试盘 (V3.0 极客版)

程序在 `bootstrap_pipeline.py` 和 `strategy_engine.py` 中定义了极其严苛的筛选标准：

| 阶段 | 指标要求 | 策略含义 |
|:----|:---------|:---------|
| **地量横盘（潜伏期）** | 过去 20 个交易日振幅 < 25% | 排除高位震荡或趋势明显的股票，确保是长期潜伏后的首次异动 |
| **主力试盘（信号点）** | 成交量 ≥ 过去 20 日均量的 **2.5 倍** | 显著放量，表明有主力资金介入测试抛压 |
| | 实体涨幅 > **2%** 且为阳线 | 确保不是无量假阳或下跌放量 |
| **防守位设定** | 标杆价 = 试盘日最高价（若上影线过长则取实体高点） | 压力位参考，突破此价位视为起爆信号 |
| | 支撑位 = 试盘日最低价 | 绝对止损底线，收盘跌破即废弃 |

### B. 状态流转机制 (The State Machine)

股票入池后，根据每日 K 线走势自动在四个状态间流转：

| 状态码 | 状态名称 | 定义与触发条件 |
|:-------|:---------|:---------------|
| **1** | **试盘池** | 刚触发倍量信号的标的，进入观察期 |
| **2** | **回踩池** | 试盘后出现极致缩量：成交量 ≤ 试盘日成交量的 50%，说明主力洗盘且未出货 |
| **3** | **突破池** | 股价收盘有效站上 **标杆价**，视为起爆信号，进入高概率攻击阶段 |
| **99** | **废弃池** | 出现以下任一情况即淘汰：<br>- 收盘价跌破 **支撑位**（形态破坏）<br>- 潜伏期超过 **25 个交易日** 仍未突破（时效过期）<br>- 洗盘期价格冲高超标杆价 8% 后回落（视为诱多出货） |

> 状态流转全自动执行，并在 `operation_log` 表中记录每一次变动，确保可追溯。

## 🗄️ 数据库架构

项目使用 SQLite 数据库，文件名为 `stock_quant.db`，核心包含以下三张表。

### 表 1：`daily_k_line`（底层行情数据）

存储全市场股票日线历史行情，是策略运算的数据源。

| 字段名 | 类型 | 说明 |
|:-------|:-----|:-----|
| `date` | TEXT | 交易日期（格式：YYYY-MM-DD） |
| `code` | TEXT | 6 位股票代码（如 `000001`） |
| `close` | REAL | 收盘价（前复权） |
| `volume` | REAL | 成交量（单位：手） |
| `amount` | REAL | 成交额（单位：元） |
| `turn` | REAL | 换手率（%） |
| `pctChg` | REAL | 涨跌幅（%） |

### 表 2：`stock_pipeline`（策略流水线）

存储当前处于策略跟踪中的股票及其关键信息，每条记录唯一对应一只股票。

| 字段名 | 类型 | 说明 |
|:-------|:-----|:-----|
| `code` | TEXT | 股票代码（主键） |
| `name` | TEXT | 股票名称 |
| `status` | INTEGER | 状态码：1-试盘，2-回踩，3-突破，99-废弃 |
| `test_high` | REAL | 标杆价（突破晋级参考价） |
| `latest_price` | REAL | 最新成交价（缓存字段，用于提升前端渲染性能） |
| `entry_date` | TEXT | 进入当前状态的日期 |
| `update_time` | TEXT | 最后更新时间 |

### 表 3：`operation_log`（审计日志）

完整记录系统的每一次自动流转与人工干预操作，确保调仓决策有据可查。

| 字段名 | 类型 | 说明 |
|:-------|:-----|:-----|
| `log_date` | TEXT | 操作日期 |
| `log_time` | TEXT | 具体时间 |
| `source` | TEXT | 触发来源：`自动处理` / `按钮处理` / `手工干预` |
| `code` | TEXT | 相关股票代码 |
| `name` | TEXT | 股票名称 |
| `detail` | TEXT | 变更详情说明 |

## ⚙️ 功能模块详解

### 1. Web 总控台 (`index.html`)

面向用户的图形化操作界面，集成了以下核心功能：

- **增量同步**：一键拉取全市场最新 K 线数据，运行状态机，完成自动化调仓。
- **三重过滤**：支持勾选“包含 ST”、“包含创业板”、“包含科创板”，实时刷新各池子计数与列表。
- **池子看板**：分栏展示试盘、回踩、突破三个核心池的股票列表，点击股票即时切换 K 线图。
- **快捷看图**：内置 ECharts 5.5.0，支持在列表或日志界面点击“📊 快捷看图”瞬间弹出交互式 K 线分析图，无需切换页面。
- **审计日志**：按日期分组展示所有策略流转记录，支持点击日志行直接调取对应股票 K 线。
- **上帝模式**：在日志界面或池子列表处，可通过下拉菜单手动修改股票状态，用于人工纠偏，操作会自动写入审计日志。
- **系统重置**：支持清空当前流水线状态，基于全量历史数据重新进行雷达回溯扫描。

### 2. 后端服务 (`server.py`)

基于 FastAPI 框架构建，提供 RESTful API 与业务逻辑核心。

- **全局任务互斥锁**：在执行“增量同步”或“系统重置”等耗时任务时，通过锁机制防止并发操作导致的数据库死锁或状态错乱。
- **BFF 模式**：针对 Web 端和移动 App 端分别提供优化接口。App 端接口经过数据预装配，实现毫秒级响应。
- **状态机引擎**：封装在 `run_strategy_engine()` 中，基于最新行情数据完成所有股票的入池、流转与淘汰判断。
- **行情数据更新**：调用 `update_daily_k_lines()` 从 BaoStock / AkShare 拉取增量 K 线。

### 3. 定时任务 (`auto_task.py`)

通过 `schedule` 库实现无人值守的自动化运行。

- **触发时间**：每个交易日 **15:35**（收盘后 5 分钟，确保数据源已更新）。
- **执行流程**：
    1. `update_daily_k_lines()` → 更新全市场最新日 K 线数据
    2. `run_strategy_engine()` → 运行策略引擎，完成状态流转
    3. `sync_data_to_app_table()` → 同步数据至 App 专用视图表

## 🔌 API 接口清单

后端服务运行于本地端口，所有接口前缀为 `/api`。

| 接口路径 | 方法 | 功能说明 |
|:---------|:-----|:---------|
| `/api/task_status` | GET | **系统体检**：查询后台任务是否正在执行，前端据此动态置灰/解锁操作按钮 |
| `/api/counts` | GET | 获取各状态池的股票数量统计（支持 ST/科创/创业板过滤参数） |
| `/api/pool/{status}` | GET | **Web 端分类查看**：根据状态码获取对应股票清单（基础字段） |
| `/api/pool/app/{status}` | GET | **App 端分类查看**：返回增强型列表（含涨跌幅、换手率等实时数据） |
| `/api/kline/{code}` | GET | **行情调取**：返回指定股票的全量历史 K 线数据，供 ECharts 渲染 |
| `/api/run_strategy` | POST | **核心动力**：手动触发增量同步与策略流转任务（受互斥锁保护） |
| `/api/update_status` | POST | **人工纠偏**：手动修改某只股票的状态，自动记录审计日志 |
| `/api/reset_bootstrap` | POST | **系统重置**：清空当前流水线状态，基于全量历史数据重新扫描 |

## 🛠️ 环境依赖与部署

### 数据源

- **BaoStock**：提供免费、稳定的 A 股历史日 K 线数据（需安装 `baostock` 并登录）。
- **AkShare**：提供全市场股票列表、概念板块等辅助信息（需安装 `akshare`）。

### 技术栈

- **后端**：Python 3.8+ / FastAPI / Uvicorn
- **定时任务**：schedule
- **数据库**：SQLite3（无需额外安装）
- **前端可视化**：ECharts 5.5.0
- **备用管理后台**：Streamlit（可选）

### 初始化步骤

1. 安装依赖包：
   ```bash
       pip install -r requirements.txt
   ```
2. 运行数据库初始化脚本：
   ```bash
    python init_db.py
    python init_basic.py
   ```
3. 启动后端服务：
   ```bash
    uvicorn server:app --host 0.0.0.0 --port 8000 --reload
   ```
4. 访问 Web 总控台：浏览器打开 http://localhost:8000/index.html
5. （可选）启动定时任务进程：
    ```bash
    python auto_task.py 
    ```
## 📌 使用注意事项

- 数据源依赖网络连接，首次运行需拉取全量历史 K 线，耗时较长，请耐心等待。
- 定时任务建议部署在云服务器或长期开机的本地机器上，以确保持续跟踪。
- 手动干预（上帝模式）时请谨慎操作，系统会如实记录所有变更，但错误的强制修改可能干扰状态机逻辑。
- 系统默认排除 ST、*ST 股票，可在 Web 界面勾选“包含 ST”以纳入观察。
